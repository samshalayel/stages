# Skill: نظام إدارة تذاكر الدعم الفني — Architecture Knowledge

## متى تستخدم هذا الملف
اقرأ هذا الملف بالكامل قبل توليد S3 أو كتابة أي كود في S4.

## الكيانات (Entities)
- SupportTicket: { id: String, studentId: String, issueDescription: String, status: String, createdAt: Date } — تمثل التذاكر المقدمة من الطلاب.
- User: { id: String, role: String, name: String, email: String } — تمثل المستخدمين في النظام بمستويات مختلفة من الوصول.

## المكونات ومسؤولياتها
| المكوّن                         | الطبقة (UI/Logic/Data) | المسؤولية الوحيدة                                       | ما لا يفعله                    |
|---------------------------------|--------------------|--------------------------------------------------|------------------------------|
| TicketSubmissionForm            | UI                 | النموذج المخصص لتقديم الطلبات                          | إدارة التذاكر                     |
| TicketManagementSystem          | Logic              | النظام لمعالجة وإدارة التذاكر                           | تقديم الطلبات                    |
| NotificationService             | Logic              | للإشعارات السريعة عبر البريد الإلكتروني            | معالجة البيانات                  |
| DataAnalysisModule              | Logic              | لتحليل بيانات التذاكر وتوليد التقارير                 | إرسال الإشعارات                  |

## القواعد الصارمة (Hard Rules)
1. HR-1: لا يجوز استخدام تقنيات غير مدعومة مثل PHP.
2. HR-2: يجب توفر واجهة المستخدم ليتوافق مع الأجهزة المكتبية والمحمولة.
3. HR-3: لا يجري استبعاد استخدام خدمات تخزين بيانات أخرى تنشأ تعقيدات.

## قرارات لا يعاد فتحها
- ADR-1: اختيار استخدام React كإطار عمل واجهة المستخدم — لتحسين تجربة المستخدم.
- ADR-2: اختيار استخدام Node.js كملحق للخادم — لدعمه السهل والنمو السلس.
- ADR-3: اختيار استخدام MongoDB — لتخزين البيانات الديناميكية وتحليلها بسهولة.

## مصطلحات المشروع (Glossary)
- SupportTicket: التذكرة المقدمة من الطلاب.
- User: المستخدمين في النظام بمستويات مختلفة من الوصول.
